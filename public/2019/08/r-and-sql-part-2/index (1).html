1. Joins

This section assumes a basic understanding of the different types of joins, specifically left join, inner join and full join. For those who are new to SQL, this section of R for Data Science, by Hadley Wickham, provides a nice explanation of the different types joins.

1.1 Inner join

Example 1

Say we want to list the name of every employee from the employees table who has one or more customers recorded in the customers table. We want to see the following information:

  • employee number
  • employee first name
  • employee second name
  • customer name(s)

This is an example of an inner join, as we only want records that have a match in both tables. In this case, we want to match the employeeNumber in in the employees table with the salesRepEmployeeNumber in the customers table.

With SQL:

SELECT employees.employeeNumber, 
       employees.lastName, 
       employees.firstName, 
       customers.customerName
FROM   employees 
INNER JOIN customers 
  ON employees.employeeNumber = customers.salesRepEmployeeNumber
ORDER BY lastName, customerName

Note that we specify the table from which column is taken, for example using employees.employeeNumber rather than just employeeNumber.

With R:

The same join could be carried out in R using the following code. In the by argument, we first list the column to match from the left hand table (‘employeeNumber’) followed by the corresponding column from the right hand table (‘salesRepEmployeeNumber’).

employees %>%   #left hand table
  inner_join(customers, by = c("employeeNumber" = "salesRepEmployeeNumber")) %>%
  select(employeeNumber, lastName, firstName, customerName) %>% 
  arrange(lastName, customerName)

Example 2

Say instead that we wanted to list every customer from the customers table who has one or more orders in the orders table. We want to see the following information:

  • customer number
  • customer name
  • order number
  • current order status

This is another example of an inner join, this time matching records using the ‘customerNumber’ column which is the same in both tables.

With SQL:

The fact that the same column name is used in both tables means we can simplify our code with USING in place of the ON expression that was used above:

SELECT c.customerNumber, 
       c.customerName, 
       o.orderNumber, 
       o.status
FROM       customers AS c 
INNER JOIN orders AS o
  USING    (customerNumber)
ORDER BY   customerName

In the example above, we’ve also used aliases in place of the full table names, i.e. ‘c’ for ‘customers’ and ‘o’ for ‘orders’. This is done with AS (though even this can be dropped). This makes the code shorter and easier to read.

With R:

The equivalent R code is shown below. Note that we now only need to specify the one column name in the by argument.

customers %>% 
  inner_join(orders, by = "customerNumber") %>%
  select(customerNumber, customerName, orderNumber, status) %>% 
  arrange(customerName)

1.2 Left join

In the example above we were interested in the records of customers in the customers table which had a corresponding order in the orders table. But what if we wanted to see records of all customers, even if they didn’t have any associated orders?

To do this we can use LEFT JOIN in SQL and left_join() in R, as follows:

With SQL:

SELECT c.customerNumber, 
       c.customerName, 
       o.orderNumber, 
       o.status
FROM      customers AS c
LEFT JOIN orders AS o
  ON c.customerNumber = o.customerNumber
ORDER BY customerName

With R:

customers %>% 
  left_join(orders, by = "customerNumber") %>% 
  select(customerNumber, customerName, orderNumber, status) %>% 
  arrange(customerName)
